# Demo Flow

## Demo Product: AirBoard

AirBoard is a fictional vibe-coded AI whiteboard collaboration app. The founder built it quickly using AI coding tools and wants to launch it publicly.

IP Breaker scans AirBoard before launch and runs IP red-team playbooks.

## Demo Script

### Scene 1: Builder wants to launch

The builder submits:

```text
Product name: AirBoard
Category: AI collaboration SaaS
Target market: US / EU / Global
GitHub repo: sample AirBoard repo
Logo: sample logo
UI screenshot: sample dashboard
Description: AI whiteboard for teams, with real-time collaboration and automated board summaries.
```

### Scene 2: Agent builds attack surface

IP Breaker identifies risk surfaces:

```text
- Source code and dependencies
- Product name and slogan
- Logo and visual identity
- UI layout
- Technical workflow and product features
```

### Scene 3: Agent runs attack playbooks

```text
Running License Contamination Attack...
Running Trademark Collision Attack...
Running Design Lookalike Attack...
Running Patent Claim Trap Attack...
Running Code Clone Attack...
```

### Scene 4: Launch Risk Dashboard

Example output:

```text
Launch Risk Score: 78 / 100
Verdict: MODIFY BEFORE LAUNCH

HIGH: Product name “AirBoard” has possible trademark collision risk in software/SaaS categories.
MEDIUM: Dashboard UI resembles common collaboration board product patterns.
MEDIUM: One dependency chain includes copyleft license obligations.
LOW: Patent search found semantically related claim clusters; FTO review recommended.
```

### Scene 5: Fix and rescan

The builder renames the product to **AeroCanvas**, changes visual styling, replaces risky dependencies, and runs IP Breaker again.

Example output:

```text
Launch Risk Score: 31 / 100
Verdict: WARN

Remaining issues:
- Related patent clusters found; human FTO review recommended before commercial launch.
- Attribution checklist should be completed before distribution.
```

### Scene 6: Casper Attestation

IP Breaker computes a scan report hash and writes a Launch Risk Attestation to Casper Testnet:

```text
work_hash
report_hash
risk_score
verdict
issue_codes
scanner_agent_id
created_at
```

The final screen shows:

```text
Launch Risk Attestation: Verified on Casper Testnet
```

## Video Goal

The demo should show that AI-built products can be created quickly, but they need an IP risk firewall before public launch.

Key message:

> AI makes products faster. IP Breaker makes them safer to launch.
